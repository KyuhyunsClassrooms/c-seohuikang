# 4. 디버깅 보고서 (10점)

> 💡 **디버깅이란?** 프로그램에서 발생한 오류(버그)를 찾아내고 고치는 과정입니다.

---

## 📝 작성 정보
- **작성 날짜:** 2025년 _11_월 _20_일
- **작성자:** 강서희

---

## 🐛 발견한 오류 #1

### 1) 어떤 문제가 발생했나요?
컴파일을 했는데 에러 메시지가 뜨고 실행이 안 됐어요.

```
문제 상황을 간단히 설명하세요.
```

### 2) 오류가 발생한 코드
```c
// printf("메뉴를 선택하세요:")

```

### 3) 왜 오류가 생겼나요?
printf 뒤에 세미콜론(;)을 안 찍어서 문장이 끝나지 않은 것으로 인식했어요

```
원인을 간단히 적어보세요.
```

### 4) 어떻게 고쳤나요?
```c
// printf("메뉴를 선택하세요:");

```

### 5) 고친 후 결과
- [ㅐ] ✅ 잘 작동해요!
- [ ] ❌ 아직 문제가 있어요

---

## 🐛 발견한 오류 #2

### 1) 어떤 문제가 발생했나요?
```
사용자가 숫자를 입력해도 프로그램이 값을 제대로 읽지 못했어요.
```

### 2) 오류가 발생한 코드
```c
// scanf("%d", choice);

```

### 3) 왜 오류가 생겼나요?
```
정수를 입력받을 때는 변수 앞에 &를 붙여야 하는데,
&가 빠져 있었어요.
```

### 4) 어떻게 고쳤나요?
```c
// scanf("%d", &choice);

```

### 5) 고친 후 결과
- [ㅐ ] ✅ 잘 작동해요!
- [ ] ❌ 아직 문제가 있어요

## 💭 디버깅하면서 배운 점

이번 프로젝트에서 오류를 고치면서 새롭게 알게 된 점이나 느낀 점을 자유롭게 적어보세요.

```
세미콜론을 빼먹으면 프로그램이 절대 실행되지 않는다는 걸 배웠습니다.
scanf로 값을 받을 때는 &를 꼭 붙여야 한다는 걸 알게되었습니다.
함수 이름은 철자가 조금만 달라도 오류가 난다는 걸 깨달았습니다.
오류 메시지를 보면 어떤 줄에서 문제가 있는지 알려줘서 도움이 된다는 걸 느꼈습니다.
```

## ✅ 최종 체크리스트

프로그램이 잘 작동하는지 최종 확인해보세요!

- [ㅐ] 컴파일이 잘 되나요? (오류 메시지가 없나요?)
- [ㅐ] 프로그램이 실행되나요?
- [ㅐ] 메뉴가 제대로 나오나요?
- [ㅐ] 입력한 값이 정상적으로 처리되나요?
- [ㅐ] 모든 기능이 잘 작동하나요?
